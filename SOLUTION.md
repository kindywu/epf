# 电力采购决策优化 — 解决方案

## 一、问题是什么

化工装置 24 小时连续运行，用电量几乎是常数。电价涨了不能停机，跌了也不会增产。但电费波动直接吞噬利润。

企业每笔购电决策都在多个时间点发生：

```
决策时刻     锁定什么           当时未知的是什么
─────────────────────────────────────────────────
每年 Q4      下一年度长协比例    明年 12 个月的月度价、日前价、实时价
每月中旬     下月度合同比例      下月每天 96 时段的日前价、实时价
每天         次日 96 时段日前量   实时价
```

**每次决策是同一个问题**：当前已知价格下，锁多少量？留多少给后续市场？

- 锁多了 → 后续市场跌价，多花钱
- 锁少了 → 后续市场涨价，暴露在风险中

这不是"预测电价"的问题。预测是手段，决策是目的。衡量标准不是 MAE，是**电费省了没有**。

### 为什么值得做

1. **供给侧变量可观测、可建模。** 电价的核心驱动力是风光出力、天气、负荷——这些都是物理变量，不是市场情绪。风电满发则边际成本接近零，无风则火电撑底、价格飙升。这种因果关系是结构性的、可捕捉的，不像股票价格由"人的预期"驱动。

2. **市场结构保留了套利空间。** 电力市场是周期性集中竞价（日前一天一次、实时 15 分钟滚动），很多参与者有刚性物理义务（工厂必须用电），没有连续交易机制让量化基金把规律抹平。这意味着电价里与天气、负荷相关的规律，比流动性好的金融市场的规律更容易持续存在。

3. **风险溢价是结构性的。** 实时市场与日前市场之间存在风险溢价——买方为规避实时波动，愿意在日前多付一点锁定确定性。承担这个波动可以换来长期期望回报，跟保险公司收保费是一个逻辑。这不需要"比别人预测更准"，只需要正确评估自己的风险承受能力。

## 二、需要哪些数据

### 必须项

| 数据 | 用途 |
|------|------|
| 历史日前统一结算点电价（P_da） | 建模标签、回测基准 |
| 历史实时统一结算点电价（P_rt） | 实时溢价建模、策略回测 |
| 历史年度/月度合同报价 | 中长期层决策优化的输入 |
| 天气数据（温度、风速、日照） | 供给侧波动的代理变量 |
| 风光出力数据 | 供给侧核心驱动变量 |
| 联络线计划 | 跨省供需变化 |
| 负荷数据 | 需求侧输入 |

### 补充项

| 数据 | 用途 |
|------|------|
| 燃料价格（煤、气） | 边际成本输入 |
| 机组检修计划、政策公告 | 供给结构变化、规则变化——非结构化文本，用 LLM 抽取 |

## 三、怎么利用数据

### 整体流程

```
数据 → 价格分布建模 → 最优锁定量 → 验证
         (分位数)     (决策函数)
```

### 第一步：价格分布建模（分位数回归，不输出点预测）

用 LightGBM/CatBoost 分位数损失函数，输出每个时段价格的分位数曲线（P10/P25/P50/P75/P90）。

两个模型：

| 模型 | 输入特征 | 输出 |
|------|----------|------|
| 日前价格分布 | 时间特征、天气、风光出力、负荷、联络线、燃料价格、历史价格滞后项 | P_da 分位数 |
| 实时溢价分布 | 同上 + 风光预测近期误差 | Δ^R = P_rt - P_da 分位数 |

**注意**：Δ^R 和 P_da 存在条件依赖——同样的风光预测误差同时影响日前价水平和实时溢价。需从同一特征集同时预测两者，避免独立建模引入依赖偏差。

### 第二步：决策优化（统一的函数，三层调用）

核心是一个函数：

```
输入：
  P_lock        当前已知的锁定价格
  P_future_dist 未来价格的概率分布（来自分位数模型）
  λ             风险偏好（越大越保守）

输出：
  α*            最优锁定量（占剩余需求的百分比）
  期望成本
  尾部风险（CVaR）

目标：min (1-λ)·E[成本] + λ·CVaR[成本]
```

三个决策层调用同一个函数，传入不同参数：

| 决策层 | P_lock | P_future_dist 来源 |
|--------|--------|--------------------|
| 年度 | 年度合同分月报价 | 月度价格分布模型 + 年前展望 |
| 月度 | 月度合同报价 | 日前价格分布模型 + 月前展望 |
| 日前 | 当日 P_da 出清价 | 实时溢价分布模型 |

### 求解复杂度：逐档验证

不一步到位用最复杂的方法。每档通过验证才升级：

- **Phase 1 — 规则基线**：完全不依赖预测模型。用历史统计定固定比例（如"冬季多锁年度，夏季少锁"）。如果这个简单规则都不能在 Walk-forward 回测中统计显著省钱，项目终止。
- **Phase 2 — 场景采样 + LP**：引入分位数预测，场景生成 + PuLP/OR-Tools 求解。必须统计显著优于 Phase 1 的固定规则才生效。
- **Phase 3 — 多阶段随机规划**（有条件触发）：嵌套 CVaR 全局优化。只在 Phase 2 已持续证明增量价值后才投入。

### 第三步：持续验证

| 指标 | 怎么算 |
|------|--------|
| 分位数校准度 | 实际值落在各分位数区间的频率是否符合预期 |
| 反事实对比 | "如果按建议做"vs"实际做法"的电费差异 |
| 显著性 | Newey-West（HAC）检验，不是普通 t 检验 |
| 漂移检测 | 滚动窗口上预测误差是否趋势性恶化 |

分级准出：影子运行 → 辅助决策 → 自动执行。每级至少一个完整季节周期验证。降级机制在漂移时自动触发。

### LLM 的四个落点

LLM 不做数值预测。用在这里：

1. **非结构化数据抽取**：从检修公告、政策文件 PDF 中抽取结构化字段（机组名称/停运时间/容量），替代脆弱的选择器规则
2. **政策影响打分**：对规则调整类文件输出"看多/看空/中性 + 置信度"作为辅助特征
3. **异常归因**：预测偏离大时，自动收集当天天气、公告、新闻，生成可能原因分析
4. **决策解释**：将优化器的数字输出翻译成人能看懂的决策理由（见下文"如何使用"）

## 四、产出物是什么

### 年度产出（每年 Q4 一次）

```
─────────────────────────────────────────────────────
  2027 年度长协采购建议

  合同报价：280 元/MWh
  建议锁定量：350,000 MWh（总用量的 70%）

  理由：
  1. 当前报价处于近 5 年同期 P35 分位 → 相对便宜
  2. 留 30% 给月度+现货：
     - 冬季（12-2月）模型预测价格大概率高于 280 → 多锁
     - 汛期（6-9月）风光出力充足，大概率低于 280 → 留到现货
  3. 期望节省（vs 全部锁年度）：~120 万元
  4. 尾部风险（最差 5% 情景）：多花 ≤ 80 万元

  分月分配建议（供谈判参考）：
  冬季 12-2月：锁 85%
  春季 3-5月：锁 65%
  夏季 6-8月：锁 55%
  秋季 9-11月：锁 70%
─────────────────────────────────────────────────────
```

### 月度产出（每月中旬一次）

```
─────────────────────────────────────────────────────
  2027 年 1 月采购建议

  年度已覆盖 85%
  1 月合同报价：300 元/MWh
  建议：不再额外锁月度，剩余 15% 走日前+实时

  理由：1 月报价高于预测日前均价（~280），不值得锁。
─────────────────────────────────────────────────────
```

### 每日产出（每天一次）

```
─────────────────────────────────────────────────────
  明日（6 月 15 日）日前采购建议

  日前出清均价：220 元/MWh
  建议：留 20% 给实时市场

  理由：
  1. 明日低温 + 高风速，风电出力大概率超日前预测
  2. 历史相似天气下实时溢价均值为负（实时更便宜）
  3. 当前实时溢价 P90 = 0.05，尾部风险可控
─────────────────────────────────────────────────────
```

## 五、用户如何使用

### 年度（每年 Q4）

1. 拿到下一年度长协合同报价
2. 将报价输入系统
3. 系统输出建议锁定量 + 分月分配 + 理由
4. 用户带着建议去谈判签合同。分月比例可作为谈判筹码（总价不变下调整分配）。**最终决策权在人手里。**

### 月度（每月中旬）

1. 拿到下月合同报价
2. 系统基于已锁仓位和最新预测，输出建议月度锁定量
3. 用户决定是否签月度合同

### 每日（每天一次）

1. 日前市场出清后，系统获取当天出清价
2. 系统输出次日"锁多少日前、留多少实时"的建议
3. 用户在日前市场按建议报价/采购

### 长期

- 每月看一次验证报告：建议 vs 实际决策的电费对比、分位数校准度、漂移检测
- 分布漂移时系统自动预警，用户决定是调整模型还是降级决策模式
