## 2. 预测目标与策略逻辑

### 2.1 核心预测对象

| 优先级 | 对象 | 符号 | 说明 |
|--------|------|------|------|
| P0 | 日前-实时价差 | `spread_da_rt = P_DA − P_RT` | 同 period，策略核心 |
| P0 | 价差方向 | `sign(spread_da_rt)` | 决定多报/少报 |
| P1 | 日前价曲线 | `P_DA[1..96]` | 报价参考 |

### 2.2 交易策略 (PnL 代理, §4.3)

简化结算公式（Q_mlt=0, Q_rt=1, 单位 MWh）：

```
电费 = Q_da × P_da + (1 − Q_da) × P_rt = P_rt + Q_da × spread
```

策略规则：
- 预测 DA < RT (spread < 0) → Q_da = 1 → **多报**（买日前）
- 预测 DA > RT (spread > 0) → Q_da = 0 → **少报**（等实时）

节省 vs 始终买日前 = (1 − Q_da) × spread

### 2.3 「容量」是什么

**容量 = 每个 15 分钟时段你经手的电功率，单位 MW（兆瓦）。**

以 100 MW 风电场为例：

```
每 15 分钟 (1 个 period = 0.25 小时)：
  发电功率 = 100 MW                              ← 这就是「容量」
  发电量   = 100 MW × 0.25 h = 25 MWh             ← 这是要交易的电量

一天 96 个 period：
  总发电量 = 100 MW × 24 h = 2,400 MWh            ← 一天要交易 2,400 度电

策略帮你对每个 period 的 25 MWh 做决策：
  - 预测日前更贵 → 目前在实时卖 → Q_da = 0 (留到实时)
  - 预测实时更贵 → 目前在日前卖 → Q_da = 1 (在日前卖)
```

**容量决定了「1.63 元/MWh 这个利润率放大多少倍」：**

| 你是谁 | 容量 | 每 period 电量 | 日交易量 | 日利润 |
|--------|------|---------------|---------|--------|
| 小型风电场 | 50 MW | 12.5 MWh | 1,200 MWh | 1,956 元 |
| 中型新能源场站 | 100 MW | 25 MWh | 2,400 MWh | 3,913 元 |
| 大发电集团 | 300 MW | 75 MWh | 7,200 MWh | 11,739 元 |
| 售电公司/省级组合 | 1000 MW | 250 MWh | 24,000 MWh | 39,129 元 |

### 2.4 日利润、年利润计算方式

#### 第一步：回测得出每 MWh 平均节省

回测在 136 天测试集上逐 period 模拟交易决策：

```
对每个 period (15 分钟)：
  if pred_spread < 0 (预测日前更便宜)：
      Q_da = 1  → 买日前 → 节省 = 0 (与 naive 相同)
  else (预测实时更便宜)：
      Q_da = 0  → 等实时 → 节省 = spread = P_da − P_rt

总节省 = Σ(所有 period 的节省) = 21,286 元/MWh-sum
平均节省 = 21,286 ÷ 13,056 periods = 1.63 元/MWh
```

这里 "元/MWh" 的含义是：**每交易 1 MWh 电量，平均比「始终买日前」节省 1.63 元。**

#### 第二步：从「元/MWh」换算到「元/天 (1MW)」

1 MW 容量运行一天 = 24 MWh 电量。因为：

```
1 MW × 24 小时 = 24 MWh
(等价于：1 MW × 0.25 小时 × 96 periods = 24 MWh)
```

所以 1 MW 一天的节省：

```
日利润 (1MW) = 1.63 元/MWh × 24 MWh = 39.12 元/天
```

用回测原始数据验证：

```
回测总节省 (1MW) = 21,286 × 0.25 MWh/period = 5,321.5 元
回测天数 = 136 天
日均节省 = 5,321.5 ÷ 136 = 39.13 元/天  ✓
```

#### 第三步：放大到不同容量

| 容量 | 每 period 电量 | 日电量 | 日利润 | 年利润 |
|------|--------------|--------|--------|--------|
| 1 MW | 0.25 MWh | 24 MWh | 39 元 | 1.4 万 |
| 50 MW | 12.5 MWh | 1,200 MWh | 1,956 元 | 71.4 万 |
| 100 MW | 25 MWh | 2,400 MWh | 3,913 元 | 142.8 万 |
| 300 MW | 75 MWh | 7,200 MWh | 11,739 元 | 428.5 万 |
| 1000 MW | 250 MWh | 24,000 MWh | 39,129 元 | 1,428.2 万 |

公式：

```
年利润 = 1.63 元/MWh × 24 MWh/MW/天 × 容量(MW) × 365 天
       = 1.63 × 24 × 容量 × 365
       = 14,282 × 容量(MW)  元/年
```

#### 第四步：关键假设

年化计算隐含以下假设，使用前需注意：

| 假设 | 说明 | 风险 |
|------|------|------|
| **策略可每日执行** | 回测仅 136 天 (2-6 月)，年化假设全年 365 天表现相同 | 未覆盖夏冬高峰季，实际可能有季节性差异 |
| **每 period 都有 1 MW 需求** | 假设每个 15 分钟时段都有恰好 1 MW 的量需要买卖 | 实际负荷有日内曲线，凌晨低、白天高 |
| **交易无摩擦** | 未考虑报价能否成交、价差小时交易成本 | 价差 <10 元时扣除手续费可能亏损 |
| **Q_da ∈ {0, 1}** | 只做二元决策（全买日前或全等实时） | 实际可以部分报量，连续优化空间更大 |
| **样本加权在真实 D-1 场景等效** | 训练用了 D-day 的 \|spread\| 做权重，实盘无法精确复现 | 可用 D-1 预测的 spread 幅度近似替代（见 §7.3） |

#### 日 PnL 波动

日均 158 元 (100MW)，但日间波动很大：

```
标准差: 1,158 元/天  (7.3× 均值)
正收益日: 43%
最佳日: +5,455 元
最差日: −4,246 元
```

这意味着策略不能只看平均——单日亏损超过 4,000 元的情况会出现，需要足够的风险准备金。

## 3. 建模方案

### 3.1 方案 A：双头回归 (最终采用)

```
D-1 特征 ──┬── DA Head (LGBM, quantile α=0.5) ──→ pred_da
           └── RT Head (LGBM, quantile α=0.5) ──→ pred_rt

pred_spread = pred_da − pred_rt
direction = sign(pred_spread)
```

### 3.2 特征工程

**DA 模型 (30 特征，继承 Andy TOP30)：**

| 类别 | 特征 |
|------|------|
| 事前预测 (7) | `ne_wind`, `hydro_fcst`, `gen_fcst`, `thermal_fcst`, `tie_*`(6选3) |
| 历史价格 (8) | `price_lag_1d/2d/3d/7d`, `price_roll_7d/30d_mean`, `price_lag_1d_dev_roll7` |
| 日历 (5) | `day_of_year`, `dow`, `days_in_solar_term`, `solar_term_sin`, `days_from_holiday` |
| 气象 (2) | `wx_temp_2m`, `wx_wind_100m` |
| 衍生/交互 (8) | `tie_total`, `supply_gap`, `coal_x_thermal_fcst`, `coal_x_net_load`, `load_fcst_x_wx_temp`, `hydro_ratio`, `price_roll_7d_mean`, `price_roll_30d_mean` |

**RT 模型 (75 特征 = DA 30 + 以下)：**

| 类别 | 新增特征 |
|------|---------|
| RT/spread 滞后 (4) | `rt_lag_1d`, `rt_roll_7d_mean`, `spread_lag_1d`, `spread_roll_7d_mean` |
| 扩展事前预测 (2) | `ne_solar`, `load_fcst` |
| 扩展气象/成本 (2) | `wx_ghi`, `coal_price_gansu` |
| Regime 标记 (4) | `is_ne_high_gen`, `is_load_peak`, `is_grid_other`, `is_solar_season` |
| 日历扩展 (7) | `is_weekend`, `is_holiday`, `month`, `is_heating_season`, `period_sin/cos` |
| 衍生 (5) | `net_load`, `renewable_ratio`, `ne_total`, `wx_wind_effective`, `wx_wind_above_cutin` |
| 贴地交互 (5) | `floor_regime_proxy`, `renewable_ratio_x_floor_regime`, `ne_total_x_floor_regime`, `net_load_x_floor_regime`, `solar_season_ghi` |
| 其他交互 (6) | `renewable_ratio_x_holiday`, `net_load_x_period_sin`, `net_load_x_holiday`, `tie_total_x_hour`, `ne_solar_per_ghi`, `renewable_ratio_x_coal` |
| 动态计算 (14) | 扩展 lag (rt_lag_2d/3d/7d, spread_lag_2d/3d/7d, rt_roll_7d_std), 价格 bucket, 交互特征 |

### 3.3 最优训练配置

```python
# DA 模型：不加权（回归保护）
LGBMRegressor(
    objective="quantile", alpha=0.5,
    n_estimators=1000, learning_rate=0.03, num_leaves=255,
    min_child_samples=20, subsample=0.7, colsample_bytree=0.7,
    reg_alpha=0.1, reg_lambda=0.1,
)

# RT 模型：按 |spread| 加权
sample_weight = clip(|spread_da_rt|, 0, 200)  # 大 spread 时段权重高
LGBMRegressor(
    objective="quantile", alpha=0.5,
    n_estimators=2000, learning_rate=0.015, num_leaves=127,
    min_child_samples=50, subsample=0.7, colsample_bytree=0.7,
    reg_alpha=0.5, reg_lambda=0.5,
)
```

## 4. 诊断发现

### 4.1 信息硬上限

| 指标 | DA 价格 | RT 价格 | Spread |
|------|---------|---------|--------|
| D-1 特征 R² | 0.78 | 0.56 | **−0.25** |
| 1 天自相关 | 0.66 | 0.57 | **0.13** |
| DA-RT 相关性 | — | — | **0.80** |

**核心结论：** DA 和 RT 高度相关 (0.80)，spread 是差值。差值信噪比仅 0.05 (|mean|/std)。D-1 特征能预测 DA 和 RT 的共同运动，但预测不了差值方向。Spread 的 R² 为负——模型不如直接猜均值。

### 4.2 误差传播分析

```
DA 预测误差 std: 107 元/MWh
RT 预测误差 std: 132 元/MWh
DA-RT 误差相关: 0.57
→ Spread 预测误差 std: 112 元/MWh
→ Spread 信号 std: 107 元/MWh
→ 误差/信号比: 1.0x
```

双头模型中两个误差部分抵消（正相关 0.57），但抵消后误差仍然和信号一样大。

### 4.3 方案对比

| 方案 | Spread MAE | 方向准确率 | 结论 |
|------|-----------|-----------|------|
| A 双头回归 | 57.5 | 54.3% | 方向最优，PnL 基线 |
| B 直接 Spread L1 | 51.9 | 43.1% | MAE 最优但方向比随机差 |
| C 分类器 (threshold=20) | — | 52.5% | 新能源大发段 77.4%，但不转化为 PnL |
| 两阶段 (pred_da 特征) | 51.9 | 53.5% | 无额外增益 |

### 4.4 天气特征消融实验

| 目标 | 有天气 R² | 无天气 R² | 差异 |
|------|----------|----------|------|
| DA | 0.696 | 0.695 | +0.001 |
| RT | 0.392 | 0.401 | −0.009 |
| Spread | −0.144 | −0.222 | +0.078 |

天气特征对价格和 spread 几乎无增量。因为 `ne_wind`、`ne_solar`、`load_fcst` 已将天气信息内化。

## 7. 结论与限制

### 7.1 已确认的结论

1. **D-1 信息集对 spread 方向存在硬上限。** Spread = 两个高度相关变量的差值，大部分方差来自 D-1 后信息。方向准确率天花板约 55%。

2. **样本加权是当前最有效的单点优化。** 不提升整体准确率，但把模型容量倾斜给大 spread 时段，PnL 从 −1.35 翻到 +1.63。

3. **天气数据对 spread 无增量。** 电力调度已将天气内化为新能源/负荷预测，原始气象冗余。

4. **模型捕获理论最大收益的 6.9%。** 剩余 93% 需要 D-1 不可得的信息（实时天气偏差、机组停运等）。

### 7.2 已知限制

- PnL 代理基于简化结算公式 (Q_mlt=0, Q_rt=1)，未考虑物理约束、中长期合约
- 回测仅 136 天（2026-02~06），未覆盖全年季节周期
- 日 PnL 波动大 (std=1,158 vs mean=158)，夏普比仅 0.14
- 模型未考虑交易成本、报价策略、省间规则

### 7.3 可能的改进方向

| 方向 | 预期收益 | 难度 |
|------|---------|------|
| 实际新能源/负荷数据 → 预测误差统计 | 中 | 高（需新数据源） |
| 跨时段特征 (ramp, 相邻 period) | 低-中 | 低 |
| Regime 分治模型 | 低 | 中 |
| 非对称损失函数 | 低 | 中 |
| 日前申报量/供需比 | 中 | 高（需新数据源） |
