# 项目复盘

> 2026-06-16

## 1. 我们究竟要解决什么问题？

**帮助化工高耗能连续生产型用户在多层电力市场上做最优购电组合，以最低成本、最稳定价格满足 24 小时不间断用电。**

化工用户核心特征：
- 24 小时连续生产，负荷极其稳定
- **不能随电价涨跌而调节生产**——电价暴涨也得用，停机损失远超电价波动
- 需要供电成本高度稳定，方便产品定价和年度预算
- 行业惯例：实时现货敞口极小（<3%），绝大部分电量提前锁定

## 2. 变量拆解

### 2.1 市场结构：不是三层，是五层

中长期合同按期限分四档，加上现货，一共五层：

```
层级      锁定时间      价格波动    灵活性        化工用户占比
──────────────────────────────────────────────────────────────
年度长协   提前1年       最低        最低（锁死）   75-90%
月度合同   提前1月       低          低            10-20%
周/旬合同  提前1周      中          中            3-8%
日前现货   前1天        高          高            <3%（极少）
实时现货   当天          最高        —            被动结算（趋近于0）
```

规则定义：只要交割周期长于日前/实时，都叫"中长期"。年度是压舱石主力。

### 2.2 决策变量（每 15 分钟时段）

| 变量 | 含义 | 决策时机 | 化工用户特征 |
|------|------|----------|-------------|
| Q_annual | 年度合同签约量 | 每年四季度签次年 | 占 75-90%，核心决策 |
| Q_monthly | 月度合同量 | 每月签次月 | 应对扩产/减产/检修 |
| Q_weekly | 周/旬合同量 | 提前 1 周 | 微调，接近现货但价更稳 |
| Q_da | 日前申报量 | 前一日 | 化工用户通常很少 |
| bid_price | 日前申报价 | 前一日 | 化工用户通常是 price-taker |

### 2.3 外生变量

| 变量 | 含义 | 不确定性 | 模型 |
|------|------|----------|------|
| P_annual | 年度合同价 | 低（年度谈判决定） | 不需要预测 |
| P_monthly | 月度合同价 | 中（与 P_da 走势相关） | 可辅助预测 |
| P_weekly | 周合同价 | 中-高 | 暂无数据 |
| P_da | 日前出清价 | **中** | ✅ LightGBM 已有 |
| P_rt | 实时价 | **高** | ❌ 无数据 |
| Q_actual | 实际用电 | **极低**（化工负荷稳定） | 用户自估，几乎是常数 |

### 2.4 结算公式（五层版）

```
电费 = Q_annual × P_annual
     + Q_monthly × P_monthly
     + Q_weekly × P_weekly
     + (Q_da − (Q_annual+Q_monthly+Q_weekly)) × P_da
     + (Q_actual − Q_da) × P_rt
```

对于化工用户，`Q_actual ≈ 常数`，`Q_rt_exposure = Q_actual − Q_da ≈ 0`。

### 2.5 历史数据

56,734 行 × 72 列，2024-11-01 ~ 2026-06-14，96 时段/天。目标变量 `price_day_ahead`。特征含负荷预测、新能源出力、联络线、天气、煤价、日历、滞后电价等。

**数据缺口**：月度合同价、周合同价、实时价的历史数据均缺失。

## 3. 可能的解决方案

### 3.1 核心洞察：化工用户的问题重心在采购组合，不在日前博弈

对比两次问题设定：

| | 之前假设（普通批发用户） | 实际（化工用户） |
|------|------|------|
| 核心决策 | Q_da：日前报多少、什么价 | Q_annual / Q_monthly：中长协组合 |
| 现货敞口 | 10-20%，可套利 | <3%，几乎不碰现货 |
| P_rt 的重要性 | 高（博弈日前 vs 实时） | **低**（实时敞口趋近于 0） |
| Q_actual 不确定性 | 中-高 | **极低**（24h 连续稳定） |
| 价格敏感度 | 追求最低成本 | 追求成本稳定 + 低价 |

**这意味着**：P_rt 缺失不再是阻塞项。化工用户不需要精细的日前-实时套利——他们需要一个好的中长协采购组合建议。

### 3.2 战略层：年度采购组合优化（最核心，现有数据就能做）

**问题**：今年四季度签明年合同时，年度/月度/周度各签多少？

**输入**：
- 用户预期用电量 Q_actual（几乎是常数曲线，化工负载 24h 稳定）
- 市场上各期限的报价：P_annual、P_monthly、P_weekly
- P_da 的历史分布（我们有 591 天数据）

**输出**：不同采购组合的期望电费和风险分布。

**分析方法**：

用历史 P_da 按 (月份, 小时) 分组，得到 P_da 的经验分布。对每种采购组合，模拟全年电费：

```
对每种 (Q_annual%, Q_monthly%, Q_weekly%) 组合:
  对历史中每一年做回测:
    模拟电费 = Q_annual × P_annual
             + Q_monthly × P_monthly
             + Q_weekly × P_weekly
             + 剩余走日前的部分 × P_da_实际
  输出: 期望电费 + P10/P90 区间
```

用户可以直观看到：多签年度（锁定低价但缺灵活）vs 多留月度（灵活但可能涨价）的成本-风险权衡。

**优点**：
- 不需要实时价数据
- 不需要未来外部特征（负荷预测等）
- 用现有 591 天历史就能给出有意义的分析
- 对化工用户，这是他们最关心的决策

**局限**：历史 P_da 分布 ≠ 未来；P_monthly 和 P_weekly 的历史数据缺失（暂时只能用它们与 P_da 的经验关系估算）。

### 3.3 战术层：日前申报（次要，锦上添花）

**问题**：中长协锁定后，每天那 <3% 的日前敞口怎么报？

**方法**：LightGBM 预测 P_da + 规则引擎。但对化工用户来说，这部分电费占比极小，决策错误的代价很低。

### 3.4 迭代路线（按价值排序，不是按时间）

```
Phase 1：年度采购组合分析（高价值，现有数据可做）
  ├─ P_da 历史分时段分布画像（月×小时热力图）
  ├─ 采购组合回测引擎（输入各期限报价，输出期望电费分布）
  └─ 分季节/分峰谷的配置建议

Phase 2：月度滚动更新（低价值，锦上添花）
  ├─ 随着月份推进，用最新数据更新 P_da 分布
  └─ 月度合同签约量微调建议

Phase 3：日前申报优化（低价值，化工用户占比极小）
  ├─ 完善 Q_da 决策引擎
  └─ 接入 P_rt 预测（如果未来拿到数据）

Phase 4：自动化（远期）
  └─ 对接外部数据源，自动化数据管道
```
