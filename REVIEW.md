### 市场规则

- 日前市场：统一边际出清，每天申报报量+报价
- 实时市场：偏差结算，缺口按实时价交易
- 买卖都不能做空
- 买上限 = 登记容量 × 120%
- 卖需要 T-2 以上锁的长协库存

### 结算公式（最新）

```
批发用户各时段电费 =
    中长期合同电量 × 合同价
  + (日前出清量 - 中长期合同量) × 日前统一出清价
  + (实际用电量 - 日前出清量) × 实时统一出清价
```

三层结算：**长协基荷 → 日前调仓 → 实时平衡**。

## 2. 目的是什么？要做什么？

**核心目标：成本最优化。**

不是一个纯预测问题，而是一个在三个市场之间的最优配置问题：

- 现货日前申报策略：每个 15 分钟时段报什么价、报多少量
- 决策依据：预期日前出清价 vs 预期实时价
  - 预期日前价 < 实时价 → 日前多买
  - 预期日前价 > 实时价 → 等实时
  - 差额不大 → 无所谓
- 卖出套利：有低成本长协库存时，日前高价时段卖出套利
- 长协基荷已经锁定了大部分用电量，日前是增量调节

**最终交付物：一个可用的日前申报决策工具。**（当前是 Streamlit Web 界面）

**最终方案**：LightGBM 分位数回归，三线输出：
- **P10**（覆盖 27%）：卖参考 — 仅 27% 实际价低于此，若 P10 > 长协成本可安全卖出
- **P50**（覆盖 67%）：中性预期 — 判断市场高低，决定买哪个市场
- **P90**（覆盖 95%）：原买入报价 — 目前已被「预期实时价」替代

### 关键发现

1. **MAPE 对买方场景是错误指标**：买方少报=买不到电，高估和低估不对称
2. **政策结构断点**：训练数据全在旧规 V3.1 下，验证/测试在新规下，制度性断裂无法通过特征弥补
3. **特征不是越多越好**：Top-30 特征（砍掉 37 个）MAPE 反降
4. **分位数回归比分点预测更适合决策**：天然给出报价上下界

## 4. 下一步做什么

基于最新的结算公式，优先级的建议：

### 短期（可立即做）

1. **修正交易逻辑** — 结算规则已明确，三层结算公式需要体现在决策中：
   - 长协电量是基荷，不参与日前报价
   - 日前申报量 = 预期用电量 − 长协已锁量（不是从 0 开始）
   - 实时偏差自动结算，不需要主动在实时市场操作

2. **决策框架升级** — 当前只比较「日前价 vs 预期实时价」，需加入长协层：
   ```
   各时段决策:
     日前申报量 = max(0, 预期用电量 - 长协已锁量)
     if 日前申报量 > 0:
       报价 = 预期实时价（如果日前出清 < 这个价就日前买，否则等实时）
     elif 预期日前出清价 > 长协价:
       卖出 = min(长协库存, 长协已锁量)（日前高价卖出长协库存套利）
   ```

3. **UI 调整** — 侧边栏加「预期用电量曲线」和「长协已锁量曲线」（96 时段），替代目前的简化参数

### 中期（需外部数据）

4. **实时价预测模型** — 目前实时价是用户手动输入的固定值。需要训练实时价预测模型（如果数据可获取）
5. **外部数据对接** — 调度系统（负荷/新能源预测）、气象服务、交易系统 API

### 长期

6. **回测框架** — 用历史数据模拟真实交易，评估策略的实际节省金额
7. **滚动训练** — 每月用新数据更新模型，适应政策变化
