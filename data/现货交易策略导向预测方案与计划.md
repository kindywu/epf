> **文档版本：** v0.2（2026-06-20）

> **关联文档：** [甘肃电力现货市场日前价预测建模方法论.md](./甘肃电力现货市场日前价预测建模方法论.md)、[domain_knowledge.md](./domain_knowledge.md)

## 1. 背景与问题重述

### 1.1 交易策略的第一性原理

不论中长期还是现货，交易逻辑可概括为：

- **预期未来价格走高 → 现在买入 / 多锁价 / 多报量**
- **预期未来价格走低 → 现在卖出 / 少锁价 / 少报量**

差异在于**交易机会次数**：

| 市场 | 机会特征 | 预测失败的代价 |
|------|----------|----------------|
| **中长期** | 交割月长、可多次交易与调仓 | 可滚动修正 |
| **现货日前** | **每个时段仅一次 D-1 申报** | 不可事后改表，误差直接进结算 |

因此，预测工作的业务价值不在于「单点价格猜得准不准」本身，而在于：

> **对同一物理时段（同一 `period`），判断不同交易时间窗口之间的相对价格走势是否正确。**

### 1.2 共识说明：方向优先、边际出清与统一价格

本节固化业务侧与建模侧的共同理解，作为后续 KPI 与方案取舍的依据。

#### 1.2.1 方向比「绝对精度」更贴近申报决策

对**价格接受者**（按全省统一出清价结算、单点体量通常不足以单独决定边际价）而言：

- **第一优先级：** 同一 `period` 上，**日前价相对实时价是偏高还是偏低**（`P_DA` vs `P_RT` 的方向），从而决定该时段**多报还是少报**。
- **第二优先级：** 日前价（及 spread）的**绝对水平与幅度**，用于报价区间、合约衔接与风险量化。

因此：

> **预测「涨/跌」在此处特指跨市场窗口的相对关系（日前 vs 实时），而非「明天 18 点比今天 18 点贵不贵」这类纯时间序列趋势。**

评估体系应**优先报告方向命中率与 spread MAE**；日前价 MAPE/MAE 仍保留，但定位为 P1 辅助指标（见 §2.3）。

#### 1.2.2 价格由边际统一出清决定，非个体报价「抬价」

甘肃现货**统一结算点**机制下：

- **同一交易场景**（如日前市场）、**同一交易时段**（如 6 月 21 日 18:00–18:15 对应的某个 `period`），全省只有一个**边际统一出清价**。
- 该价格由**供需曲线相交、边际条件**决定，**不是因为**某一市场主体的报价「报得太高」就把全省价格抬高。
- 预测任务本质是：在 D-1 信息集下，推断 D 日各 `period` 的**市场均衡出清价**，以及日前窗口与实时窗口两个出清价的**相对关系**。

本方案中的「用户」视角默认为**价格接受者**。若主体体量极大、常处于边际，则报价会影响出清——属做市/大电源逻辑，不在 Phase 1 默认范围内。

#### 1.2.3 同一 period、不同窗口：两个统一价

| 维度 | 示例 | 价格个数 |
|------|------|----------|
| 市场窗口 | 日前（DA）vs 实时（RT） | 各有一套出清价 |
| 物理时段 | 6/21 18:00 第一个 15 分钟（某 `period`） | 每个窗口 **1 个**全省统一价 |
| 策略决策 | D-1 申报该 `period` 的日前电量 | 在预估 **统一** `P_DA` 与 **统一** `P_RT` 下选择多报/少报 |

**真正需要预测的核心对象**是：`spread_da_rt = P_DA − P_RT`（同 period），及其符号所代表的申报方向。

#### 1.2.4 边界说明（避免过度简化）

1. **方向正确 ≠ 一定高收益：** 当 `|P_DA − P_RT|` 很小时，方向对但经济意义弱，故除方向准确率外仍需 spread 幅度与 **PnL 代理**（§4.3）。
2. **绝对水平并非无用：** 中长期衔接、报价上下限、风险敞口仍依赖对 `P_DA` 水平的合理估计——现有 Andy 日前价模型继续作为 **P1 基础设施**。
3. **MLT 价不单独决定日前报量方向：** 在 DA–RT 关系给定下，中长期主要影响结算水平；与 §1.3 结算结论一致。

**一句话共识：**

> 在统一结算点现货中，每个 period 的价格由市场边际出清决定；交易者预测的是「日前与实时两个窗口谁高谁低」，**方向比单点绝对误差更贴近申报决策**；现有日前价预测是必要输入，但不是策略的充分目标。

### 1.3 现货日前申报的核心结论（结算视角）

在「中长期 + 日前 + 实时」三段式结算下（简化）：

- **日前申报量方向**（多报 / 少报）主要由 **`日前价 vs 实时价`（同 period）** 决定：  
  - 日前 **高于** 实时 → 倾向 **少报**（日前卖出更划算）  
  - 日前 **低于** 实时 → 倾向 **多报**（日前买入更划算）
- **中长期合约价** 主要影响**结算金额水平**与中长期头寸盈亏，**不单独决定**日前申报方向（在 DA–RT 关系给定的情况下）。

> 上述逻辑是策略层的简化规则；实际申报还需考虑物理约束、风险敞口、省间规则与合约分解，本方案先建立**可回测的价差与方向指标**，再接入完整申报优化。

### 1.4 与当前项目的关系

| 层级 | 当前状态 | 新目标 |
|------|----------|--------|
| 数据 | 日前价、五项事前信息、天气、煤价；实时价抓取能力已有 | 日前 + 实时 **对齐宽表**，可选接入 MLT 合约价 |
| 模型 | Andy Oracle 预测 **D 日日前价 96 点** | 增加 **DA–RT 价差 / 方向** 预测与策略 KPI |
| 评估 | MAE / sMAPE / MAPE；贴地 & 西北三时段 regime | **方向命中率、价差 MAE、结算 PnL 代理、相似日基线** |
| 产出 | `predictions_batch_*.csv`、宽表 | **申报建议视图**（96 点方向 + 置信度 + 价格曲线） |

**原则：不推翻现有 pipeline**，在 D-1 信息边界与 96 点网格不变的前提下**扩展标签、评估与产出**。

## 2. 目标定义

### 2.1 预测对象（按优先级）

| 优先级 | 对象 | 符号 | D-1 是否可推理 | 策略用途 |
|--------|------|------|----------------|----------|
| **P0** | 同 period **日前 − 实时价差** | `spread_da_rt = P_DA − P_RT` | 训练用 D 日真值；推理用模型 | **日前申报方向** |
| **P0** | 价差 **方向** | `sign(spread_da_rt)` 或 `P_DA > P_RT` | 同上 | 多报 / 少报分类 |
| **P1** | D 日 **日前价曲线** | `P_DA[1..96]` | 现有 Andy | 报价参考、曲线形态 |
| **P1** | D 日 **实时价曲线**（或 RT 期望） | `P_RT[1..96]` | D-1 仅模型估计 | 与 P0 等价（知 DA+RT 即知 spread） |
| **P2** | 日前 vs 中长期 | `P_DA − P_MLT` | 需 MLT 曲线数据 | 曲线衔接、套保 |
| **P3** | 实时 vs 中长期 | `P_RT − P_MLT` | 事后复盘 | 最终盈亏归因 |

**Phase 1 聚焦 P0 + P1**；MLT 相关放 Phase 3（依赖合约数据接入）。

### 2.2 决策时点与信息边界（继承方法论 §1.2）

- 所有特征仍为 **D-1 及更早**可得：平台事前预测 A–Q、Historical Forecast 气象、煤价（D-1 对齐）、价格 **滞后**（含 D-1 日前/实时）。
- **禁止**在 D-1 推理时使用 D 日真实实时价、实测负荷或 ERA5 气象。
- 训练时可使用 D 日 `P_DA`、`P_RT` 作为标签；评估必须模拟 D-1 推理路径。

### 2.3 成功标准（业务 KPI）

| 类别 | 指标 | Phase 1 目标（待基线后修订） |
|------|------|------------------------------|
| **方向** | 全样本 DA>RT 方向准确率 | 显著优于 50% 与相似日基线 |
| **方向** | 分时段（西北三时段）方向准确率 | 新能源大发段、高峰段分别 > 基线 |
| **幅度** | `spread_da_rt` 的 MAE（元/MWh） | 优于「spread 历史同期均值」基线 |
| **水平** | 日前价 MAE | 不劣于现有 Andy（回归保护） |
| **策略** | 结算 PnL 代理（见 §4.3） | 优于 naive「全报实际量」或相似日策略 |
| **形态** | 逐日曲线 Pearson / 峰谷时刻偏差 | 方法论 §6.3 逐步纳入 |

> **MAPE 不作为主 KPI**（贴地日失真）；与方法论及现有 regime 评估一致。

## 4. 建模方案

### 4.1 总体架构

```
┌─────────────────────────────────────────────────────────┐
│  D-1 特征（宽表 + lgbm_features 链）                      │
└───────────────────────────┬─────────────────────────────┘
                            │
         ┌──────────────────┼──────────────────┐
         ▼                  ▼                  ▼
   ┌───────────┐    ┌───────────────┐   ┌─────────────┐
   │ Head-A    │    │ Head-B        │   │ Head-C      │
   │ P_DA      │    │ P_RT 或 spread│   │ P(DA>RT)    │
   │ (现有Andy)│    │ (新增)        │   │ (分类,可选) │
   └─────┬─────┘    └───────┬───────┘   └──────┬──────┘
         │                  │                   │
         └──────────────────┼───────────────────┘
                            ▼
              ┌─────────────────────────┐
              │ 策略层：96 点申报建议      │
              │ direction + confidence  │
              └─────────────────────────┘
```

**Phase 1 推荐路径（低风险）：**

1. **保留 Andy 预测 `P_DA`**（已有）。
2. **新增 spread 模型** `P_RT − P_DA` 或直接 **`P_RT`**（二选一，见 §4.2）。
3. **方向** 由 `pred_spread` 符号导出，或单独训轻量分类器。

### 4.2 候选建模方案对比

| 方案 | 做法 | 优点 | 缺点 | Phase |
|------|------|------|------|-------|
| **A. 双头回归** | 共享特征，两路 LGBM：`P_DA`、`P_RT` | 可解释；与现有 Andy 兼容 | 两模型维护 | **1** |
| **B. Spread 单目标** | 直接回归 `spread_da_rt` | 直接对齐策略 KPI | 贴地/限价处 spread 分布偏态 | **1** |
| **C. 分类 + 回归** | 先 `P(DA>RT)`，再回归 \|spread\| | 方向准 | 两阶段复杂 | 2 |
| **D. 序列模型** | Seq2Seq 联合输出 DA+RT 曲线 | 形态好 | 数据量有限 | 3 |

**建议：Phase 1 采用 A（双头）+ 基线 B 对照**；A 的 spread = pred_DA − pred_RT 自然得到。

### 4.3 策略层与结算 PnL 代理

简化结算（与用户提供的申报策略表一致，单位：元）：

设某 period：

- `Q_mlt`：中长期分解电量（常数或已知）
- `Q_da`：日前申报量（策略变量）
- `Q_rt`：实际电量（常数，如 400）
- `P_mlt, P_da, P_rt`：对应价格

```
电费 = Q_mlt·P_mlt + (Q_da − Q_mlt)·P_da + (Q_rt − Q_da)·P_rt
     = Q_mlt·(P_mlt − P_da) + Q_da·(P_da − P_rt) + Q_rt·P_rt
```

在 `Q_rt` 固定时，**最优 Q_da 方向**由 `P_da − P_rt` 符号决定（忽略约束时：DA>RT → 少报；DA<RT → 多报）。

**回测 PnL 代理（Phase 1）：**

- 固定 `Q_rt = 1`（单位 MWh）归一化；
- 策略：`Q_da = 1` 若 pred_spread < 0，否则 `Q_da = 0`（或连续化版本）；
- 与 oracle（真值 spread 符号）、naive（Q_da = Q_rt）对比 **单位电量结算成本**。

### 4.4 特征与后处理

- **继承** 现有宽表 + `lgbm_features`（lag、regime、西北三时段、贴地后处理）。
- **新增 RT 相关 lag**（同 period 历史实时价、历史 spread）。
- **RT 头** 可复用贴地/高峰 regime 后处理（实时同样存在贴地与跳涨）。
- **训练样本截断：** walk-forward；train 不含测试日；与 `model_split.yaml` 对齐。

### 4.5 基线（必须）

| 基线 | 定义 | 用途 |
|------|------|------|
| **相似日 spread** | 近 N 天同星期 + 相近净负荷日的 spread 同时段均值 | spread MAE / 方向 |
| **相似日 DA** | 方法论 §5.1 | 保护现有 DA 水平 |
| **持久性** | spread_lag_1d 同 period | 最低门槛 |
| **Oracle 方向** | sign(真实 spread) | 理论上界 |

### 5.1 CLI / 报告扩展（规划）

|--------------|------|

6. **典型失败日：** 如 2026-06-19（DA 高估 + spread 方向错误分解）

## 7. 风险与依赖

| 风险 | 缓解 |
|------|------|
| 实时价历史缺口大 | 优先 backfill；评估仅用交集日 |
| RT 比 DA 更难（D-1 信息更少） | 双头 + spread lag；接受 RT MAE 高于 DA |
| 策略简化与真实申报不一致 | Phase 1 只做方向 KPI；Phase 2 引入约束 |
| 模型增多维护成本 | 共享特征链；统一 `evaluate-*` 入口 |
| MLT 数据不可得 | Phase 3 可选，不阻塞 P0 |

## 9. 附录

### 9.1 术语

| 术语 | 含义 |
|------|------|
| DA / 日前 | Day-ahead，D-1 申报、D 日出清 |
| RT / 实时 | Real-time，D 日实际运行 |
| MLT / 中长期 | 合约市场，交割月长 |
| spread | 本方案默认指 `P_DA − P_RT`（同 period） |
| period | 15 分钟时段，1–96 |

### 9.2 西北电网三时段（评估维度）

| 时段 | 时间（常规西北） | 策略含义 |
|------|------------------|----------|
| 新能源大发 | 10:00–16:00 | 贴地、spread 结构特殊 |
| 用电高峰 | 06–09、17–22 | 抬价、RT 偏差风险 |
| 其他 | 其余 | 过渡 / 夜间 |

（已实现特征：`is_ne_high_gen`、`is_load_peak`、`is_grid_other`。）
